## E04：为什么人眼检测有±0.5mm误差

**时长**: 30-40秒
**BGM**: 科技感，数据弹出时配合音效

### 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-3s | 手指放在一把尺子上，指甲盖宽度 ≈ 1mm | "你的指甲盖厚度，大概一毫米" |
| 3-6s | 标注画面：一个产品，标注尺寸公差要求 ±0.1mm | "但很多电子元件的公差，要控制在0.1毫米以内" |
| 6-10s | **分屏**: 左边人拿卡尺测量，读数在跳动；右边OasysCoreCV亚像素测量画面，数值精确到μm | "卡尺读一次变一次——而我们的亚像素算法，精确到微米" |
| 10-15s | OasysCoreCV系统画面：测量结果「∅ 5.023mm」，显示精度 0.001mm | "千分之一毫米的精度。" |
| 15-20s | 列出3个测量值对比：卡尺 5.0mm / 千分尺 5.02mm / OasysCoreCV 5.023mm | "卡尺、千分尺、我们的系统——谁更准？" |
| 20-25s | 动画：像素→亚像素定位原理示意（cornerSubPix） | "原理不复杂——但能做好，需要工程积累" |
| 25-30s | 结尾 LOGO + 文字「你们测量用什么工具？」 | "你们产线现在还用手工测量吗？" |

## E05：SIFT vs 模板比对，哪个更准？

**时长**: 30-40秒
**BGM**: 理性对比风

### 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-3s | 一块产品在检测台上，正位置放好 | "模板比对，是AOI最基础的检测方式——" |
| 3-6s | 模板比对运行画面，正常检测通过 | "只要你的产品放在正确位置，就OK" |
| 6-10s | 产品旋转了30°，再跑模板比对→**FAIL**，误报一堆缺陷 | "但如果产品旋转了——" |
| 10-15s | **对比**: 同样旋转角度，SIFT特征匹配运行→所有特征点正确匹配→PASS | "换SIFT特征匹配——旋转？缩放？它不在乎" |
| 15-20s | 动画展示SIFT特征点：旋转前后相同特征点自动配对 | "它找的是特征点，不是像素。" |
| 20-25s | 统计对比卡：模板比对/误报率30%/SIFT/误报率2% | "哪个更准？数据说话" |
| 25-30s | 结尾 LOGO | "这就是为什么我们集成了8种检测模式——不同场景，不同算法" |

## E06：FFT频域分析——周期性缺陷的克星

**时长**: 35-45秒
**BGM**: 前半段疑问→后半段"原来如此"感觉

### 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-3s | 一块有周期性纹理的PCB/屏幕面板 | "有些缺陷，在图像里是藏起来的" |
| 3-6s | 传统模板比对扫描一遍→结果无异常 | "普通检测方法——什么都没发现" |
| 6-10s | 动画：图像→傅里叶变换→频域图（heatmap） | "但如果把图像转换到频率域——" |
| 10-15s | 频域图出现异常亮点（非正常频率成分） | "这些不该出现的频率成分——就是缺陷" |
| 15-20s | OasysCoreCV 频域分析运行画面：异常频率自动标出 | "我们的系统自动做傅里叶变换，找出异常频率" |
| 20-25s | 频域缺陷标注→反变换回图像→3处缺陷标红 | "再看原图——3个缺陷，清清楚楚" |
| 25-30s | 适用场景图标：PCB线路、屏幕面板、布料纹理、太阳能电池 | "PCB线路、面板、布料、太阳能电池——只要是周期性纹理，FFT就是最强手段" |
| 30-35s | 结尾 LOGO | "这就是企业级检测和普通检测的差距" |
