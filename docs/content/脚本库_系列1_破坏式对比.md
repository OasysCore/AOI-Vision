## E01：人工目检 vs AI检测，同一天的产品

**时长**: 30-45秒
**BGM**: 前半段轻松日常，缺陷检出瞬间切换紧张鼓点

### 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-3s | 工厂产线全景快切，质检员拿着放大镜在工作 | "一个质检员，一天最多看2000个零件——" |
| 3-5s | 特写：质检员揉眼睛，打了个哈欠 | "到下午3点，眼睛已经花了" |
| 5-10s | **分屏画面**: 左侧质检员在显微镜下找了个缺陷花了30秒，右侧AOI系统3秒同时标出3个缺陷 | "但我们这套系统，从来没累过" |
| 10-15s | 右侧OasysCoreCV系统运行画面，连续检测多个工件，每个＜1秒出结果 | "一秒一个，一秒一个——它不眨眼的" |
| 15-20s | 数据面板弹出：今日检测12000件，检出缺陷47个 | "今天跑了12000件，抓到47个缺陷" |
| 20-25s | 画面切回质检员+文字弹窗「漏检率？」 | "这47个里，有12个是人工目检根本没发现的" |
| 25-30s | 结尾：OasysCoreCV LOGO + 文字「想知道你们产线的漏检率？↓评论区聊聊」 | "想知道你们产线的漏检率？评论区告诉我" |

- [ ] OasysCoreCV 系统运行录屏（检测画面+结果面板）

## E02：1000个零件，人工数 vs AI数

**时长**: 25-35秒
**BGM**: 紧张倒计时感，AI完成瞬间音效炸裂

### 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-3s | 桌上铺满密密麻麻的零件，画面震撼 | "1000个零件——" |
| 3-6s | **快放**：一个人低头一个一个数，计数器逐渐增加 | "人工数一遍，最快也要3分钟" |
| 6-8s | 人工数到一半，数字卡住，重新数 | "数到一半忘了，再来" |
| 8-12s | **分屏**: 右边OasysCoreCV画面，镜头一扫，瞬间完成计数 | "机器呢？" |
| 12-16s | OasysCoreCV系统画面特写：计数结果「1000/1000」，耗时「2.8秒」 | "2.8秒。一个不差" |
| 16-20s | 并列对比数据卡：人工 3分钟 误差±3个 / OasysCoreCV 2.8秒 误差0 | "3分钟 vs 2.8秒。这就是差距" |
| 20-25s | 结尾 LOGO + 文字「你的产线还在靠人数？」 | "你的产线还在靠人数？" |

### 拍摄清单

- [ ] OasysCoreCV 计数功能录屏
- [ ] 快放/缩时拍摄人工计数过程

## E03：这台PCB肉眼看着完美

**时长**: 35-45秒
**BGM**: 悬疑氛围，层层递进

### 脚本

| 时间 | 画面 | 旁白/字幕 |
|------|------|----------|
| 0-3s | 手拿着一块PCB板，肉眼查看 | "这块板子，肉眼看着——" |
| 3-5s | 镜头推进到板面特写 | "完美。没问题" |
| 5-8s | 切到放大镜视角，能看到一些微小痕迹 | "放大再看看——好像有点痕迹？" |
| 8-12s | 切到显微镜视角，某个焊点周围有不明显的裂纹 | "显微镜下——这个焊点有裂纹，不仔细看根本发现不了" |
| 12-18s | **OasysCoreCV 频域分析画面**: 傅里叶变换热力图 → 异常频率点自动标注 → 共标出3处缺陷 | "但我们的系统，直接上频域分析。傅里叶变换一跑——" |
| 18-22s | 3处缺陷逐一放大标注：焊点裂纹、线路氧化、阻焊层破损 | "3个问题。肉眼一个都没看到" |
| 22-28s | PASS/FAIL 判定画面 + 缺陷详情表 | "这就是为什么出货前，最后一关必须是机器" |
| 28-35s | 结尾 LOGO + 文字「你会用肉眼检测PCB吗？」 | "你会用肉眼检测PCB吗？" |
